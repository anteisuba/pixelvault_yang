# 01. 产品范围与第一版边界

## 产品一句话

`Personal AI Gallery` 是一个面向创作者的 AI 图片生成平台。用户可以在同一个界面里切换不同模型和 Provider，生成图片、保存作品、发布到公开画廊，并获得 Prompt 模板推荐。

## 目标用户

- 想快速比较不同模型效果的创作者。
- 想保存自己作品和 Prompt 资产的个人用户。
- 想找灵感、参考模板、快速出图的轻度设计用户。

## 这个产品真正解决什么问题

- 现在用户常常要在多个平台之间来回切换，体验割裂。
- 各家模型的参数、风格、价格、结果质量不统一，不容易比较。
- 用户会不断重复写 Prompt，却没有一个地方把自己的高质量经验沉淀下来。
- 新手想要“某种风格的图”，往往不知道 Prompt 怎么写。

## 我想先挑战你两个假设

### 假设一：先做支付再说

不建议。

原因很简单：你现在还没验证用户到底会为了什么付钱。现在就做支付，工作量大、决策多、返工概率也高。更聪明的做法是：

- 第一版先做免费 Beta。
- 用积分、限额、邀请码来控制成本。
- 同时把账本和套餐结构预留好。

### 假设二：把用户 Prompt 收集起来，热门的直接推荐

这个方向有价值，但不能直接照搬。

风险在于：

- 很多用户不希望自己的原始 Prompt 被别人看到。
- 原始 Prompt 往往很长、很脏、包含私密上下文，不适合直接推荐。
- 如果直接抄热门 Prompt，产品会很快变成“Prompt 搬运站”。

更好的第一版做法是：

- 默认只把 Prompt 作为用户自己的历史记录保存。
- 只有在用户同意的前提下，才把 Prompt 用于推荐分析。
- 推荐给别人的内容优先做成“模板”和“意图卡片”，而不是直接公开原文。

## 第一版必须有

- 登录后生成图片。
- 3 到 5 个模型可切换，覆盖不同风格。
- 用户自己的历史作品库。
- 公开画廊，用户可以选择是否发布自己的作品。
- Prompt 模板推荐区。
- 基本的积分、限流、错误反馈、失败重试。
- 可以部署上线的环境配置和监控能力。

## 第一版不要有

- 完整订阅和支付体系。
- 原生手机 App。
- 社交功能，比如关注、评论、私信。
- 团队协作、多租户。
- 让用户自己配置 Provider Key 的 BYOK 模式。

## 推荐的第一版用户路径

1. 用户打开 Landing Page。
2. 看到平台支持哪些模型、适合什么风格。
3. 注册或登录。
4. 进入 Studio，输入 Prompt，选择模型，生成图片。
5. 生成结果自动进入个人作品库。
6. 用户可以把某张图发布到 Gallery。
7. 用户在推荐区看到热门模板或相关 Prompt 结构，继续迭代生成。

## 项目复杂度

整体复杂度：中等偏上。

原因不是前端难，而是这个项目同时包含：

- 外部 API 编排。
- 成本控制。
- 用户数据和 Prompt 数据。
- 公开发布时的稳定性与风控。

## 你需要准备的资源

- 域名和部署平台。
- PostgreSQL 数据库。
- 对象存储。
- 身份认证服务。
- 至少 2 到 3 家图片模型 Provider 的正式 API Key。
- 错误监控和产品分析工具。

## 我建议的发布策略

- 先做公开可访问的 Web Beta。
- 默认移动端可用，但不单独做手机 App。
- 等 Web 跑顺了，再考虑 PWA 或原生封装。
