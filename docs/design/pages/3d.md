# 3D Page

Last updated: 2026-06-02

This document records current page-level facts for the Studio 3D workspace. It
is not a redesign spec and not a request to change UI code.

### Route Surface

| Surface   | Route        | Current UI entry    | Notes                    |
| --------- | ------------ | ------------------- | ------------------------ |

### Structure

Route behavior:

- generates metadata from `Model3DGenerate`
- checks Clerk auth on the server
- signed out: renders `Studio3DWorkspace` with empty initial generations
- signed in: ensures DB user and prefetches image generations for source
  selection

Current visible structure:

- workspace header
- main 3D canvas / viewer area
- source image or generated model preview
- `ModelViewer` / `WireframeModelPreview`
- source picker/upload controls
- model picker through `MainModelPicker`
- provider key/setup prompts
- Rodin/FAL-specific controls and multiview controls
- generation action controls and result download/AR actions

## Page CSS / Layout Rules

Current CSS facts:

- root workspace uses `h-[calc(100svh-5.75rem)]` and `md:h-svh`.
- page uses shadcn semantic primitives: `bg-background`, `bg-card`,
  `border-border`, `bg-muted`, `text-muted-foreground`.
- no broad `3d-*` class family was found.
- many dense controls use `text-[10px]`, `text-[11px]`, viewport height caps,
  and max-width/max-height utilities.

## Interaction Details

Current page-internal interaction matrix:

| Interaction                  | Current trigger / owner                                 | Current state / feedback                                                                                         | Design notes                                                                                    |
| ---------------------------- | ------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
| Signed-in source preload     | `/studio/3d` route server data                          | Signed-in users receive initial image generations for source selection                                           | Source picker depends on image-only preload facts.                                              |
| Deeplink source hydration    | `?source=<generationId>` in `Studio3DWorkspace`         | Fetches the generation once and sets it as source image                                                          | Keeps Image Studio to 3D Studio handoff intact.                                                 |
| Select source asset          | Source picker / `AssetSelectorDialog`                   | Locks picker to image assets and sets `sourceImage`; clears model result when source changes                     | Source image and result model are intentionally separate states.                                |
| Select existing 3D model     | Existing-model picker / `AssetSelectorDialog`           | Locks picker to `model_3d` assets and hydrates previous GLB generation                                           | Existing model open should not require a new source image.                                      |
| Upload source image          | Upload button / hidden file input                       | Validates image upload, preprocesses through `prepareImageUpload`, uploads via `uploadImageAPI`, shows uploading | Upload path creates a source, not a finished 3D generation.                                     |
| Clear source                 | Source clear action                                     | Clears source image and related source-derived state                                                             | Must not accidentally delete uploaded/generated asset records.                                  |
| Main model selection         | Shared `MainModelPicker`                                | Updates selected 3D model; locked/key-required model can open `QuickSetupDialog`                                 | Provider/BYOK setup is part of the workspace control plane.                                     |
| Quick setup                  | Locked model or setup prompt                            | Opens `QuickSetupDialog`; completion can unlock/select provider-backed model                                     | Design must preserve provider key boundary and not imply free generation.                       |
| Preset selection             | 3D preset controls                                      | Applies bundled model/multiview/PBR/face-count settings                                                          | Presets change multiple hidden knobs; selected preset must remain understandable.               |
| Rodin mode selection         | Rodin mode segmented control                            | Switches image/text/mesh-first generation path; text mode can generate from prompt without source image          | This is a major branch in form requirements.                                                    |
| Extra Rodin angle references | Rodin angle upload controls                             | Uploads/removes additional angle images with per-angle uploading state                                           | Manual reference images are bounded and should remain visibly tied to Rodin workflows.          |
| Source quality guard         | Derived source quality checks                           | Blocks generation when source is too small or extreme-aspect; shows translated quality warnings                  | Guardrail is design-critical because it explains disabled submit.                               |
| Multiview generation         | Auto/manual 4-view controls                             | Generates side views from current source/model; supports forced regenerate and manual upload/remove per view     | Multiview is temporary provider output and should not be confused with final archived GLB.      |
| Advanced provider controls   | Provider-specific switches/selects/sliders/inputs       | Updates dense Hunyuan/Trellis/Rodin options such as PBR, face budget, mesh mode, texture mode, and quality knobs | Needs grouping hierarchy; do not flatten all controls into equal visual weight.                 |
| Generate model               | Main generate button                                    | Calls model-3D generate API; tracks stage/progress/upload progress; disables while generating or invalid         | Submit readiness depends on selected model, source/prompt mode, source quality, and key status. |
| Wireframe preview            | `WireframeModelPreview` for generated/preview model URL | Loads GLB through Three.js, displays percent/loading label, and has error label                                  | Preview is visual evidence, not the final model-viewer control surface.                         |
| Model viewer                 | `ModelViewer` / `ModelViewerInner`                      | Registers `<model-viewer>`, auto-rotates by default, supports camera controls and slotted AR button              | Browser/custom-element behavior needs real device/browser QA.                                   |
| Poster upload on visibility  | `model-viewer` `model-visibility` event                 | Calls optional poster upload callback once the GLB becomes visible; failure is non-fatal                         | Poster upload is polish/storage metadata, not generation success.                               |
| Download / AR result actions | Result controls inside model viewer/result panel        | Exposes GLB download and AR button when available                                                                | AR capability is platform-dependent; document QA should separate app bug from browser support.  |
| Retry with new seed          | Compatible generated model result action                | Re-submits generation with same model/source path and new seed intent                                            | Preserve cost/key implications in copy and disabled states.                                     |
| Refine with Hunyuan          | Triposr/compatible result action                        | Re-submits with Hunyuan target using source/reference URL                                                        | This is a cross-model continuation, not simple retry.                                           |
| Continue with textures       | Rodin mesh-first result action                          | Starts Rodin texture continuation from mesh-first output                                                         | Texture continuation is a second generation step and needs explicit state/cost design.          |
| Keep as final                | Result action for temporary/final candidates            | Marks current candidate as final in local workspace state                                                        | Should be visually distinct from persisted asset publishing.                                    |
