# Design Reviews Structure

Last updated: 2026-06-02

This directory will hold UI audits, screenshot reviews, and before/after notes.

Do not use this directory as a permanent dumping ground.
Only keep reusable reviews that help future UI work.

Review docs should include:

- Scope
- Screenshots or routes checked
- Issues
- Severity
- Proposed direction
- Validation method
- Follow-up decisions
