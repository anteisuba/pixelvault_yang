## 二、Phase 0 完成后的 Studio 架构

### 2.1 顶层布局（三模式完全一致）

```
StudioPage
└── StudioWorkspace
    └── StudioFlowLayout
        ├── canvas   = <StudioCanvas />          ← 结果预览区
        ├── dock     = <StudioBottomDock />      ← 提示词 + 参数
        └── gallery  = <StudioGallery />         ← 历史侧栏
```

**不再有 video 短路分支**。`state.outputType` 切换只影响子组件内部渲染，不改变上层结构。

### 2.2 StudioFormState 结构（video 相关）

```ts
// src/contexts/studio-context.tsx
interface StudioFormState {
  outputType: 'image' | 'video' | 'audio'
  selectedOptionId: string | null // 跨模式时自动重置（见 StudioPromptArea）
  prompt: string
  aspectRatio: AspectRatio

  // Video 专属（v3 新增）
  videoDuration: number
  videoResolution: string | null
  longVideoMode: boolean // 当前 UI 未暴露
  longVideoTargetDuration: number // 当前 UI 未暴露

  // 通用
  advancedParams: AdvancedParams // negativePrompt 在此
  panels: PanelName[] // 新增 'videoParams'
}
```

### 2.3 三模式分叉点汇总

| 位置                          | image                    | video                      | audio                  |
| ----------------------------- | ------------------------ | -------------------------- | ---------------------- |
| `StudioPromptArea` buildInput | `buildImageInput()`      | `buildVideoInput()`        | `buildAudioInput()`    |
| model options hook            | `useImageModelOptions`   | `useVideoModelOptions`     | `useAudioModelOptions` |
| `StudioToolbarPanels` 按钮    | enhance / ref / ar / adv | enhance / ref / ar / video | voice / clone          |
| `StudioCanvas` lastGeneration | outputType === 'IMAGE'   | outputType === 'VIDEO'     | outputType === 'AUDIO' |
| `useUnifiedGenerate` entry    | `generateImage()`        | `generateVideo()`          | `generateAudio()`      |

### 2.4 生成链路（与 v2 一致，未动）

```
StudioPromptArea.handleGenerate
  → useUnifiedGenerate.generate({ mode, image|video|audio })
  → POST /api/studio/generate
  → generation.service.ts
  → provider adapter (fal / volcengine / fish-audio / gemini / ...)
  → R2 upload + Prisma 持久化
  → 通过 StudioGenContext 回写 lastGeneration
  → StudioCanvas 渲染
```
