## 0. 定夺（TL;DR）

**范式 = 「装配手势 + 两级墨线」——手是吞，账是线。**

- **吞噬保留为唯一绑定手势**：拖拽入卡 / 快投点选 / 卡匣再喂，全部现状保留；不恢复端口拖线。
- **连线回归为关系台账**，分两级：
  - **骨干边**（镜头图→视频、视频→视频、一切→片盒）= 制片流，**常显**中性墨线；
  - **成分边**（角色/场景/音色/参考视频/剧本→目标）= 供给关系，默认藏进成分栏 chip，**选中节点时显石绿细线**（即 §6 探针的正式化，石绿落点③ 已覆盖，不新增颜料落点）。
- **折叠面收窄确认**：只有素材原子（散图）被吃后折叠（现状已如此），收集器卡 / 镜头图卡 / 视频卡 / 片盒永远在桌上；被折叠者始终以成分栏缩略 chip 存在，可拆出回桌。
- **动效诚实分流**：会消失的（散图）保留三拍吞噬；不消失的（镜头图→视频等）改「墨线签署」——落卡后本体弹回原位，一条墨线从来源画到目标 + chip 落定。动画与结果一致，不再"吞了却还在"。
- 节点交互同步统一：全类型一套点击模型 + 统一选中工具条解剖 + 详情三档 + 覆盖层阶梯（§3/§4，回答任务书 ❓A–E）。

## 1. 为什么这么定（三选项评估）

| 维度                  | A 全连线做好                | B 深补救吞噬（嵌套可见等）     | **本案：装配 + 两级墨线**                             |
| --------------------- | --------------------------- | ------------------------------ | ----------------------------------------------------- |
| 直观性（owner 的痛）  | ✅ 最直接                   | ⚠ 学习成本仍在（"去哪了"要教） | ✅ 制片流一眼可读，成分按需展开                       |
| 画布干净度            | ❌ 散图/音色全回桌 + 满屏线 | ✅                             | ✅ 折叠与 chip 保留，线只画少数骨干                   |
| 差异化                | ⚠ 变回"又一个节点画布"      | ✅                             | ✅ 差异化靠墨线/纸卡/盖章的身份，不靠"没有线"         |
| 语义诚实（装配≠流动） | ❌ 线暗示流动               | ✅                             | ✅ 手势仍是装配；线是"谁参照了谁"的台账，不是管道     |
| 与已建资产兼容        | ❌ 卡匣/成分栏/三拍动画贬值 | ✅                             | ✅ 全部复用（§6.1 清单）                              |
| 工程成本              | 中（路由/避障/捆绑）        | 高（嵌套容器新范式）           | **低**（渲染层一个条件 + 边视觉扩一档，数据层零改动） |
| 可回退性              | 单向                        | 差                             | **双向**（一行条件回全隐藏；再一行回全显示）          |

四个关键判断：

1. **owner 的痛 = 关系可见性丢失，不是容器化本身**（4.8 诊断成立）。吞噬为干净/差异化牺牲了"关系恒显可追溯"，修复应该补可见性，而不是推翻装配语义。
2. **差异化不必靠"没有线"**。`brand-dna.md` 画布房间的专属组件草案本来就写着「场记卡节点 · **墨线连线** · 剧本笺助手 dock · 蓝图 minimap」——墨线本就是这间房的手艺，回归连线不是回归 ComfyUI。
3. **两级是谱系上的正确点**。全显示（A）把 S5a–S5f 建成的卡匣/收集器/成分栏/三拍动画整体贬值，且散图杂讯回桌；全隐藏（现状）把制片叙事一并抹掉。画布的图天然浅（素材→镜头→视频→片盒，深度 ≤3），骨干边数量少、方向一致（左→右），不会意大利面。
4. **工程上近乎免费**。数据层边从未删除（[StudioNodeWorkbench.tsx:1666](../../src/components/business/node/StudioNodeWorkbench.tsx) 无条件 `hidden: true` 是唯一开关）；边组件 `NodeWorkflowStatusEdge` 四态视觉、合法性矩阵 `node-connection-rules.ts`、`--node-edge` token 全部健在。
5. **业界证据（2026-07-17 补充调研，全文见 `references/ui-inspiration/node-canvas-edge-paradigms-2026-07.md`）**：九个主流工具（Krea Nodes / ComfyUI / Fal / Invoke / Figma Weave / Flora / Houdini / UE / FigJam）**零先例默认全隐连线**；同时没有工具靠全量裸显取胜（Fal 因裸露被批乱），收敛答案是"分层"——**Houdini 与 v3 逐条同构**：wires（生产主流）常显且永不可隐，dependency links（参考关系）分"不显示/仅选中/全部"三档，方向纪律入左出右。UE Hide Unrelated（4.23 起）验证"选中驱动的关系聚焦"是一线专业功能。

**判读锚（owner 拍板用）**：R3-1 落地后打开一个含 镜头→视频→片盒 的项目——若"直观"回来了，范式成立继续 R3-2+；若仍别扭，说明痛在范式本身，回退成本 = 渲染条件一行（回全隐藏）或走 A（回全显示），数据无损。

## 2. 关系呈现 v3 规范

### 2.1 两级边的判定（纯函数，新增 `src/lib/node-edge-tier.ts`）

以 (sourceType/sourceRole → targetType) 判定，与 `node-connection-rules.ts` 同层放置、同风格测试：

| 边                                                | 级       | 理由                                  |
| ------------------------------------------------- | -------- | ------------------------------------- |
| 任意视频源 → `videoMerge`                         | **骨干** | 成片进片盒 = 制片主线                 |
| `seedance` → `seedance`                           | **骨干** | 前片引用（reference-to-video）        |
| 镜头图（`shot` 或 `image` role=shot）→ `seedance` | **骨干** | 静帧先审再喂 = 中鱼主线               |
| 角色/场景（收集器或 legacy）→ 镜头图 / `seedance` | 成分     | 视觉参考，成分栏 chip 已表达          |
| `voice` → 角色 / `seedance`                       | 成分     | 听觉身份 / 旁白 donor                 |
| `videoReference` → `seedance`                     | 成分     | 运动/风格参考，非制片主线             |
| closeup → 角色                                    | 成分     | 特写子参考                            |
| `shotText` → `seedance`                           | 成分     | 剧本笺直填方向，节点本身退役中        |
| 散图 → 任意                                       | 成分     | 折叠路径，通常无线可画（见 2.2 守卫） |

规则收敛为两条：`target === videoMerge` 为骨干；`target === seedance && source ∈ {seedance, 镜头图}` 为骨干；其余成分。

### 2.2 渲染条件矩阵（取代现状"无条件 hidden"）

一条边渲染，当且仅当 **两端节点都可见**（折叠源无线，由成分栏 chip 代言）且命中下表任一行：

| 条件                               | 骨干边     | 成分边                   |
| ---------------------------------- | ---------- | ------------------------ |
| 默认                               | ✅ 常显    | ❌ 藏（chip 代言）       |
| 任一端节点被选中（含框选并集）     | ✅         | ✅ **显现**（石绿细线）  |
| 目标节点生成中                     | ✅（脉冲） | ✅（脉冲，进行中靠动效） |
| 「关系」总开关开启（§2.5，可选项） | ✅         | ✅（中性墨线）           |
| 快投模式进行中                     | ✅         | 维持现状高亮体系，不叠加 |

### 2.3 墨线视觉态（更新 `resolveNodeWorkflowEdgeVisual` + `NODE_STUDIO_EDGE_VISUALS`）

| 态                           | 颜色                                                           | 宽  | 其他                                  |
| ---------------------------- | -------------------------------------------------------------- | --- | ------------------------------------- |
| 骨干默认                     | `--node-edge`（#6f6a63 中性墨）                                | 2   | smoothstep；无 glow；端标墨点         |
| hover                        | `--node-edge-active` 提亮                                      | 2.5 | 现状机制保留                          |
| **显现**（选中节点的关联边） | `color-mix(in oklab, var(--node-paint) 80%, var(--node-edge))` | 1.5 | "石绿细线"，落点③（选中态）覆盖       |
| **边自身被选中**             | `--node-paint`                                                 | 2.5 | Del/Backspace = 解绑                  |
| running                      | `--node-edge-active` + 脉冲                                    | 2   | 现状 `.node-canvas-edge-running` 保留 |

- **端标**：目标端一枚 r≈3.5 的墨点（自定义 SVG marker，fill 随 stroke），不用箭头——方向由左进右出的节点解剖 + 左→右流向表达，箭头太"流程图"。
- **石绿纪律核对**：显现/选中边属于 `node-canvas.md` §5 落点③「选中描边（节点/连线选中态）」，**不新增落点**；骨干默认是中性墨，不占颜料。
- 现值 strokeWidth 3 / previewStrokeWidth 3.5 是"仅预览时代"的宽度，常显后降为 2 / 2.5，避免线抢卡。

### 2.4 端点与端口：锚点化（调研加固，2026-07-17）

- Handle 数据层保留（ReactFlow 边定位依赖），**视觉全部退场**（CSS 透明），边直接落在卡缘 + 墨点端标收尾。
- `isConnectable=false` 全局：**不恢复端口拖线**，绑定只走吞噬/快投——两套绑定手势并存必然混乱。
- 端口类型色 5 支（含 `-on-paper` 变体）**停止渲染**，token 保留不删（iconPlate 等卡面用途不受影响）。类型身份由卡片头图标表达，不由端口表达。
- **证据**（详见 `references/ui-inspiration/node-canvas-edge-paradigms-2026-07.md` §2③）：保留类型色端口的工具（Krea/ComfyUI/Fal/Invoke/UE）保留它只因**拖线是它们的绑定手势**；绑定不走端口的白板系（FigJam）一致选择无常驻端口、按意图显现。残余信号的精确先例 = ComfyUI Hidden 模式"只有已连接端点保留彩点"——即本案"墨点只出现在有可见边处"。类型合法性反馈由拖拽张口/摇头承担（已实现，比色配对更强）。

### 2.5 「关系」总开关（调研后荐案：**保留，双态**）

底部工具条 zoom 组后加一枚 toggle（lucide `Waypoints`，zh「关系线」/en Relations/ja 関係線，aria-pressed）：开启 = 全部两端可见的边以中性墨线显示（诊断/复盘用），会话级状态默认关。

- **先例**（调研 §2②）：Houdini 依赖线三档菜单 / ComfyUI Canvas 菜单临时隐藏连线 / UE Hide Unrelated 工具栏钮——"关系显示档位"是行业标准 affordance。
- **明确不做三态**（不提供"全隐"档）：Houdini 从不允许隐藏 wires，只让参考层分档；骨干边=制片流，若可全隐，「关系可追溯」承诺退化为可选项，正好退回 owner 痛点。双态 = 默认档 ↔ 全显档，够用且不添模式负担。
- 这是 2026-07-10「透视模式也不做」拍板的定点复活——当时前提（连线彻底退场）已被本案修订。

### 2.6 折叠规则（谁消失、谁留下）——回答 ❓E

维持代码现状（S5d 后已收窄），本案予以**确认为长期契约**：

- **折叠**：散图（loose image）被吃（`fusedIntoNodeId` 或有出边）→ 折叠进目标成分栏，缩略 chip 常驻，拆出到零引用自动回桌（派生规则，零持久化字段）。
- **不折叠**：收集器卡（角色/场景）、镜头图卡、视频卡、片盒、音色、参考视频——被引用也留在桌上（卡匣是镜像托盘，不是唯一表面）。
- "消失"重新表述为**"上桌归档"**：素材原子从散落桌面归档进卡片图集/成分栏，痕迹（chip + 可拆出）永远在；产出物（能生成的东西）永远可见。

### 2.7 手势与动效（分流）——R3-2

手势总表不变（拖拽吞噬 / 快投点选 / 卡匣再喂 / 成分栏×拆出）。动效按"结果是否消失"分流：

| 来源                                                | 落卡动效                                                                                                                                                                                                                                      | 依据                                                                                    |
| --------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------- |
| 折叠源（散图、卡匣引用副本）                        | **三拍吞噬现状保留**（张口/吸入/消化落定）                                                                                                                                                                                                    | 它真的消失进卡，动画诚实                                                                |
| 非折叠源（镜头图→视频、视频→片盒、收集器→镜头图等） | **墨线签署**：目标轻咽（scale 0.98→1.02→1，240ms）+ 墨线从来源锚点 stroke-dashoffset 画入目标（~320ms，`--ease-ingest` 沿用）+ chip 落定弹（现状 340ms）；**被拖本体弹回拖拽起点**（~300ms 软回弹，`--ease-soft-return`），不留在目标卡上叠着 | 它不消失，"吞了却还在"是现状的语义裂缝；画一条墨线 = 手作签署动作（brand-dna 动效性格） |
| 咬不动                                              | 现状保留（红描边 + 摇头 + 软弹回 + 原因气泡）                                                                                                                                                                                                 | 大声暴露不动                                                                            |
| 解绑（chip × / 选中边 Del）                         | 墨线反向褪去（dashoffset 反放 ~240ms）+ chip 消失；折叠源拆出仍走反向飞回                                                                                                                                                                     | 与签署对称                                                                              |

`prefers-reduced-motion`：画线/褪线降级为淡入淡出（全局兜底已有）。张口 hover（scale 1.08 + 石绿 outline）与磁吸/快投三件套不动。

### 2.8 骨干边的创建与删除闭环

- 创建：吞噬建边（现状唯一数据变更路径）→ 若判定为骨干边，签署动画后**常显**。
- 删除：三路等价——成分栏 chip ×（现状）/ 详情面板取出（现状）/ **选中边按 Del**（新增，`edgesFocusable` + 删除键映射，走同一 removeEdge 路径，零引用自动回桌规则不变）。
