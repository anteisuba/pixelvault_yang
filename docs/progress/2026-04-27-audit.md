## 1. 项目一句话理解

PixelVault 是一个**多模型 AI 创作工作站 + 永久归档平台**：

- BYOK 模式（Bring Your Own Key），用户自带 HuggingFace / Gemini / OpenAI / FAL / Replicate / NovelAI / VolcEngine / Fish Audio 八家 provider 的 API key
- 三种生成形态：image / video / audio，统一在 `/studio` 路由下
- 所有产物永久落 R2 + Postgres，可在 Gallery / Profile / Arena / Storyboard / Collection 重复消费
- 已上线社交层（Like / Follow / Collection / Creator Profile / Arena ELO）

**栈**：Next.js 16 (App Router + Turbopack) · TS · Clerk · Prisma 7 + Neon · Cloudflare R2 · next-intl (en/ja/zh)

**规模实测**：

- 79 个 service 文件，154 个测试文件
- 60 个 Studio 组件，49 个 hooks
- 8 个 provider adapter（全部已有单测）
- Studio 三层 context split（Form/Data/Gen），驱动 23+ 组件

## 5. Studio 现状（重点）

### 5.1 实际代码 vs 文档差异

**`docs/plans/frontend/studio-feature-map.md` 已有部分过时**：

1. **VideoGenerateForm.tsx 不存在** — 该文档（Last updated 2026-04-13）的 Primary files 表里仍把 `VideoGenerateForm.tsx` 列为视频入口，但实际 grep 整个 src 找不到这个文件。视频参数现在通过 `StudioVideoParams.tsx`（在 Studio dock panel 里）处理，与 image/audio 共用 canvas shell。

2. **三栏布局描述与实际不符** — `unified-development-plan.md` B1 写的是「左栏(280px) | 中心(flex-1) | 右栏(320px)」，但实际 `StudioWorkspace.tsx` + `StudioFlowLayout.tsx` 用的是 **Canvas + Dock + Gallery 垂直流式布局**：

   ```
   [SidebarProvider Sidebar]
     [TopBar]
     [WorkflowGroupTabs + Picker + Summary]  ← workflow shell（Phase 3 新增）
     [StudioFlowLayout]
       [Canvas (flex-1)]
       [BottomDock (shrink-0)]
       [Gallery (在折叠下方)]
   ```

   `StudioLeftColumn` / `StudioCenterColumn` / `StudioRightColumn` 这些组件**不存在**。文档写错了。

3. **Studio Gallery filters 已实现** — `studio-feature-map.md` 写「`StudioGallery` filter tabs `all / favorites / today` 是 UI-only」，但 `StudioGallery.tsx:130-142` 真的有 filter 逻辑，favorites 用 likedIds.has() 过滤、today 用 ISO 日期前缀比对。`Heart` 按钮也接了 `useLike` hook（`StudioGallery.tsx:126`）。所以这条已经不再 partial。

4. **预览操作 Super Res / Remove BG / Save 已经接入** — `studio-feature-map.md` + `current-status-audit.md` 都写「currently disabled」，但 `GenerationPreview.tsx:399-431` 三个按钮都有 `onClick={() => void handleImageEdit(...)}`，`disabled` 是条件性的（`editingAction !== null`），不是永久禁用。

## 8. 一句话结论

**项目不缺功能，缺的是「让现有功能站得稳」的收口动作。**

PlanA 已经把核心管道、Studio shell、image-transform、worker execution 全部铺到位 — 但：

- 真相源文档落后代码 1-2 周，未来任何 Studio 改动会基于错误事实做决策

- workflow shell 第 6 phase polish + 双真相决策没做完
- 计费语义未对齐，阻塞货币化

PlanB 里**最高 ROI 的不是新功能，是把上面这些先收掉**。功能向继续推可以等收口完成后照「unified-development-plan §后续优先级重排」表执行。
