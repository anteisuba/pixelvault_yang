# Product 参考 — 产品边界与主线（长期契约）

> 沉淀自 `archive/product/{scope,mainline}.md`（2026-06 拍板），2026-07-10 与 owner 重核并更新四项（用户定位 / 北极星 / 3D / Arena·Storyboard）。方向不明确时停下问 owner，不由 AI 补全路线。

## 一句话定位

**以创作控制为核心的个人 AI 创作工作台**：多模型生成 + 永久归档 + 资产复用，先做创作闭环，再选择性公开展示。公开社区、排行榜、社交网络不是当前主定位。

## 用户定位（2026-07-10 更新）

**双轨并行**：自用为主（owner 是第一用户），但重要功能同时按创作者标准打磨。长期目标是降低门槛服务一般用户，但不为此削弱创作者工作台的控制力。

## 北极星（2026-07-10 更新：双核）

**画布（长视频导演台）+ LoRA（还原 + 定制）是双核**；其余域允许拓展和优化，但不与双核抢资源。图片/声音/卡片是给双核供弹药的能力层。

第一主路径（默认保护，冲突时优先）：

```text
选择模型 → 输入 prompt/素材 → 生成 → 持久保存 → 管理/复用 → 可选公开
```

路线冲突时的优先级：①创作控制与生成成功率 ②持久归档与可复用资产 ③模型/provider/API 正确性 ④Studio 主创作体验 ⑤资产化闭环（Assets/Prompts/Cards/Gallery）⑥公开展示与社交。

## 3D（2026-07-10 重定位）

- **现阶段：搁置**——不投入新能力开发。
- **远期方向（计划阶段，未到执行）**：3D 建模作为**导演台的视频控制手段**——用 3D 场景/机位控制视频镜头制作。届时 3D 的价值不是产出 3D 资产本身，而是给画布长视频提供空间控制。
- 遗留未决（低优先级挂起）：3D 结果在 Gallery/Profile 的同级展示、GLB/poster 生命周期。

## 已确认方向要点（详版见 archive/product/mainline.md）

- **图片工作台原则**：默认轻量（prompt/模型/参考图/比例/生成），专业能力 chip 渐进披露，不拆新手/专业双模式；生成后必须有去向（变体/编辑/参考/转视频/存 Recipe/存 StyleCard/发布/入 Project）；批量管理归 Assets，不塞回 Studio。
- **StyleCard 资产化**：结构化风格定义 + 参考样张 + 模型适配 + 参数快照 + 结果反馈回流；轻度/严格两档应用强度。
- **声音**：VoiceCard 独立资产身份，可绑 CharacterCard；`Character → Voice → Script → Shot/Video` 在画布连接；不做未授权声优复刻。
- **视频分工**：Studio Video = 快轻短片；画布 = 长视频/系列镜头/角色一致性/声音绑定/分镜/参考视频/片段合并。一致性来自卡片+镜头规格+参考约束的组合，不靠单一 prompt。
- **执行架构**：Worker-first（Next.js 只做 auth/validation/job/dispatch/callback）；已拍板细节见 `backend.md` / `providers.md`。

## 开发规则（长期有效）

1. 不为 UI 包装牺牲 provider/API 正确性。
2. 不把 image/video/audio/3D 执行层强行合并成统一抽象。
3. 不把 Assets/Gallery/Prompts 的职责塞回 Studio。
4. 不把公开展示做成默认主线。
5. 模型/provider/payload 改动前必须重查官方资料，不凭记忆。
6. 非 trivial 实现先确认 task packet。
7. 方向不清楚时停止问 owner。
