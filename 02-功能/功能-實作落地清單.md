| 文件                                                     | 類型     | 做什麼                                                                         |
| -------------------------------------------------------- | -------- | ------------------------------------------------------------------------------ |
| `src/services/image-transform.service.ts`                | **新建** | `transformImage(input)` 主入口；Strategy Pattern 按 `transformation.type` 路由 |

**關鍵實作要點**：

```ts
// image-transform.service.ts
export async function transformImage(input: TransformInput): Promise<TransformOutput> {
  // auth + validation 已在 api-route-factory 層做

  switch (input.transformation.type) {
    case 'style':
      return handleStyleTransform(input)

    case 'pose':
    case 'background':
    case 'garment':
    case 'detail':
      throw new NotImplementedError(
        `${input.transformation.type} coming in Phase 2+`,
        { i18nKey: 'Transform.errors.notImplemented' }
      )
  }
}

// handle-style-transform.ts 內部流程
async function handleStyleTransform(input): Promise<TransformOutput> {
  // 1. 編譯 Style Card/Preset → prompt + provider + advancedParams
  const recipe = await compileRecipe({
    styleCardId: input.style.cardId,
    presetId: input.style.presetId,
    ...
  })

  // 2. 4 variants 並行（F17 寬容）
  const N = input.variants  // 1 or 4
  const results = await Promise.allSettled(
    Array.from({ length: N }).map(() =>
      generateImageForUser({
        prompt: recipe.compiledPrompt,
        modelId: recipe.modelId,
        referenceImage: input.subject.imageData,  // i2i path
        advancedParams: {
          ...recipe.advancedParams,
          preservation: input.preservation,      // 作為 preservation hint
          strengthFromMode: mapPresetToStrength(input.preservation),
        },
      })
    )
  )

  // 3. 按成功數計費（F17）
  const successCount = results.filter(r => r.status === 'fulfilled').length
  await chargeCreditsByTransaction(userId, successCount)

  // 4. 組裝 TransformOutput
  return {
    original: { ... },
    variants: results.map(mapResultToVariant),
    totalCost: successCount,
  }
}
```

```ts
// 偽代碼
await db.$transaction(async (tx) => {
  for (const [idx, result] of results.entries()) {
    if (result.status === 'fulfilled') {
      await tx.apiUsageLedger.create({
        userId,
        generationJobId: parentJobId,
        variantIndex: idx,
        cost: 1,
      })
    }
    // failed variants：不寫 ledger
  }
  await tx.generationJob.update({
    where: { id: parentJobId },
    data: { status: 'COMPLETED', totalCost: successCount },
  })
})
```

## 6. 禁止事項（Anti-pattern）

- ❌ 在 `image-transform.service` 重寫 provider / R2 / 計費 / retry 邏輯（違反 F9 + F13）
- ❌ 為 pose / background 等新維度**新建獨立 service**（違反 F13；必須走 `image-transform/handle-*.ts` 目錄內）
- ❌ 跳過 `transformation.type` / `subject.type` 的 TS union 直接硬寫 `style`（違反 F14 抽象預留）
- ❌ 把 Preset 硬編碼在前端組件（必須走 `constants/transform-presets.ts` 單一來源）
- ❌ Phase 1 實作 Face API 檢測（F8 明確延後，用 consent modal 兜底）
- ❌ Phase 1 實作 OCR + region preservation（F7 明確延後）
- ❌ Phase 1 實作 Subject Card 建檔 UI（F12 延後）
- ❌ Phase 1 實作獨立 `/transform` 頁面（F19 拒絕 RF17）
- ❌ 批量 variants 失敗時全部回滾計費（違反 F17：按成功數扣）
- ❌ 靜默重試失敗格（違反 F17：必須用戶手動 Retry）
- ❌ 每次生成前 modal 提示成本（違反 F16：用主操作旁 toggle）
- ❌ 在 `components/business/studio/` 外的地方引入 `StudioInputImage`（此組件**專屬 Studio**）
