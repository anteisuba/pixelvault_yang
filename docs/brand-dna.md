## 一句话主基调

**「工坊宅邸。」** PixelVault 是一栋创作者的手作宅邸：一个暖纸 + 炭墨 + 手作装帧的材质世界；十个房间（十个功能域）各有隐喻、颜料与专属组件。**每页 UI 和组件都要有自己的特色，但都长在同一栋宅子里。**

## 气质关键词

有温度的手作感 · 私人画册/档案馆 · 每间房有自己的手艺 · 专业密度不打折 · 一看不像 AI 模板 · 不模仿任何现有产品

## 世界层（全屋共享；= 未来 frontend.md 共同文档的方向框架）

- **材质**：暖纸底 + 炭色墨（方向；取代 v1 的纯深/纯白双面）。
- **字体方向**：zh 要有真正的标题字身份（明朝体/衬线系）+ 手写批注字点缀 + 现代无衬线正文。**具体选型待字体对比板拍板（待深挖）。**
- **装帧语言**：裱框 / 贴入 / 盖章 / 标签纸 / 藏书票。
- **动效性格**：手作动作（贴上、翻页、盖章）取代机械过渡；仍只服务状态澄清 / 连续性 / 反馈，不做无意义装饰。**细则待深挖。**
- **交互底线（继承 v1，不随方向变）**：焦点环 · 键盘可达 · 44px 触达区 · ResponsiveOverlay 披露 · 空态起手势 · 不支持不渲染 · 触屏软键盘只在直接点输入框时弹。

## 颜料纪律（P0 红线，已拍板）

1. 颜料只给六个创作域；档案四间（画廊/提示词/素材/角色卡）不执颜料，让藏品发色。
2. 颜料只出现在「该房间的工具痕迹」上：导航徽标 / 进度 / 生成态 / 空态插画 / 选中描边。
3. **永远不做大面积彩色底色**；一屏一支颜料，不混支。

## 工程气质（这个代码库的性格，与视觉方向无关、不随改版变）

1. **长期建模优先** — 每个改动按「整体 + 日后版本」权衡；本属「属性」的东西别编码成「类型」；宁可多花成本走对架构。
2. **失败大声暴露** — 不「静默成功」；两种模式冲突时明确选一种不折中；不确定性 surface 出来。
3. **代码即事实源** — 文档与代码冲突信代码；文档只记录代码读不出的契约、决策、验证路径。
4. **复用大于重造** — 改之前先 grep exports 和调用方；已有 80% 的引擎别重写。
5. **确定性交给代码，判断交给 LLM** — 路由/重试/状态码/确定性转换写代码；只把判断型任务给模型。
6. **完成必须可核对** — 报告 = 改动清单 + 验证结果 + 手动验证步骤 + 图示；不要只说「已完成」。

## 过渡期规则（重要）

1. 新方向**尚未落地任何页面**。某页按房间文档完成重设计前，该页的日常 UI 改动仍匹配**代码现状约定**（v1 双面模式 / 反相 CTA / 无彩灰阶 / 圆角阶梯 / 动效刻度，见 `archive/design/direction.md`），**不许单点混入新风格**。
2. 改版路径固定：页级工作流调研 → `references/pages/<页>.md` 施工图 → 概念稿核对 → 实施（Fable 出施工图，Sonnet 执行）。
3. 全局字体 / 材质切换是独立地基工程，单独立项，不随单页顺带改。
