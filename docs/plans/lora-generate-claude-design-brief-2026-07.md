## 0. 一句话任务

在**结构已实现并经确认**的「LoRA Generate」页上，出「中性深炭工作台」的**高保真视觉方案**（域级/页面级 token 与质感层）。**结构不动，只做视觉。**

## 1. 产品与页面背景（自包含，冷启动可读）

- **PixelVault** 是个人 AI 创作工作台：生成图/视频/音频、永久归档、资产复用。全站是暗色应用壳。
- **LoRA Generate** 把用户已选的 LoRA、底模、来源证据、参考图、Prompt 和真实生成参数，组织成一次可检查、可迭代的出图任务。唯一主工作流：**当前装配 → 生成输入 → 出图 → 结果**。
- **现状**：页面结构已经实现、可真机运行；但视觉只是用 shadcn 标准暗色 token 临时拼的，**缺一层 LoRA 专属的深炭工作台质感**。你的任务就是补这一层。

## 2. 怎么起（用 web capture）

- 抓现有实现页：在浏览器登录态下打开 `http://localhost:3000/zh/studio/lora?section=generate`（或生产站对应页），用 web capture 抓「**已挂载 LoRA + 助手关闭 + 有参考图 + 有结果**」这个主态。
- web capture 只用来拿**真实的结构、内容和布局关系**当底子——**不要继承它当前的皮肤**。现状皮肤是临时标准暗色，不是答案。

## 3. 锁死不可动（结构层，已 owner 逐项确认）

方向 = **A「并排监视台」**。以下一律不动一个像素，你只在其上换质感：

- **顶部**：模式导航「生成 / 库 / 训练」→ 一条**无卡片装配行**（当前 LoRA · 权重 · 容量 N/max · 底模身份 · 执行通道 · 助手入口）→ **来源图带**（已挂载才出现，横向效果证据；未挂载整条消失）。
- **左 60%（输入区，自然文档流）**：参考图大卡（Prompt 上方，横排 + ＋添加 + 参考强度）→ 搭配状态条（单行，点击原位向下展开）→ **Prompt（200–240px 主编辑面，绝对主角）** → Negative Prompt（默认折叠）→ 参数 disclosure（收起显摘要）→ **出图（自然流单动作，非 sticky，只出现一次）**。
- **右 40%（结果监视列）**：大结果图（绝对主角）→ 图下元信息（尺寸 · Steps · Seed · 当前装配）→ 本次会话缩略历史（生成 >1 张才出现）。
- 主线顺序、60/40 比例、各区归属、动作位置 = 全部固定。

## 4. 你要探索的（视觉 / token 层）

1. **表面层级**：中性深炭的**多级表面梯度**——工作区底 / 装配面 / 输入面 / 结果监视面各自的炭灰值与相互关系。现状全糊在一两层上，缺纵深，这是最大的提升点。
2. **分隔与容器**：用**发丝规线和边界**表达结构层级；克制的容器语言。**不要** card/pill 堆叠、不要玻璃/发光/渐变、不要装饰性 chrome。
3. **排版**：UI 字体表达；元信息与数值（尺寸 · Steps · Seed · 权重）的排版层级与可读性——这是工作台的“数据感”所在。
4. **交互态**：hover / focus / selected / disabled，以及主动作（出图）的视觉层级；loading / error / empty 的表达。
5. **密度与节奏**：工作台该有的紧凑但不拥挤的间距节奏。
6. **参照方法**：借鉴 **Claude.ai / ChatGPT 产品工作台**的任务聚焦、排版主导、克制容器和按需控件——**借方法，不复制品牌皮肤**；不是营销官网，不是聊天结构，不是模型市场。

## 5. 明暗与色（已确认基调）

- **中性深炭工作台**：柔和炭灰 + 中性层级。避免纯黑压迫、蓝黑大浮层、发光渐变、高饱和强调。
- 强调色克制：主动作（出图）用中性高对比（现状是白丸暗字），不引入花哨主色。

## 6. 提取项目设计系统（提脊柱，不提皮肤）

- 让你读 codebase 提取的是**语义脊柱层**：`--background / --foreground / --card / --muted / --muted-foreground / --border / --input / --ring / --primary` 系（oklch；暗色 `--background`≈14.5% 深炭、`--foreground`≈98.5%、`--muted-foreground`≈70.8%、`--border` 白/0.1）+ 半径梯度 `--radius: 0.625rem`（sm/md/lg/xl 派生）。
- **不要**提取或套用：全站 `ui/` 组件的默认 card/pill 皮肤、其它业务域（Canvas / Studio / Assets）的域级 token。你产出的深炭质感是 **LoRA 域专属**，将来会落成严格作用域在 `.lora-*` 根的域/页 token，**不外溢全站**。

## 7. 红线（硬约束，违反即打回）

- 不采用被否候选「标本索引」的骨白纸底 / 朱砂 / 等宽数据声语言（那是无权重草稿）。
- 不 card/pill 堆叠、不玻璃 / 发光 / 渐变、不纯黑、不蓝黑浮层、不高饱和强调。
- 不复制 Claude.ai 的品牌皮肤。
- 结构 / 内容 / 交互一个像素不动——**你改皮不改骨**。
- 域级 token 严格 LoRA 作用域，不提升为全站默认。

## 8. 产出格式（方便交回工程侧落地）

- **主态高保真** + 几个关键状态证明同骨架成立：未挂载 · 搭配展开 · 助手停靠（右侧约 380px）· 空结果 · loading / error。
- **一份可读的 token 建议**：表面梯度、发丝规线、文本层级、焦点、主动作、半径、间距、密度的具体值（oklch 优先，贴项目命名习惯）。
- 导出 HTML / URL 交回工程侧：由工程侧（Sonnet）落成 `.lora-*` 作用域的 domain/page token（改 `globals.css` 作用域块 + 组件 className），**不重写已实现的结构组件**；再过 `checklists/ui.md` + 真机三验（浅色误用 / NSFW 底 / 域切换 color-scheme）。

## 附：三个标志性组件（视觉重点落这三处）

1. **装配与来源证据头** — 一行读完当前 LoRA、底模、执行身份，下一行用来源图证明效果。
2. **参考图 + Prompt 输入栈** — 大参考图承担视觉判断，Prompt 保有稳定创作空间。
3. **稳定结果监视列** — 大图持续可见，元信息可核对，会话历史轻量切换。
