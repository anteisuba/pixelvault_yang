### Library 三方向共同输入（已确认）

为了确保比较的是结构，而不是内容质量，三个方向固定使用同一份页面状态：

- 删除“工作台”独立标题栏；顶层直接呈现 Generate / Library / Train 工作模式，当前为 Library。
- 使用同一组搜索词、来源/归属/类型/底模筛选条件；类型与底模必须直接可见或一眼可达，不能同时藏入没有当前状态摘要的通用筛选按钮。
- 使用同一组 6–10 个 LoRA 结果及其相同样例图和元数据。
- 都必须覆盖默认浏览态，以及一个 LoRA 被选中后打开详情的状态。
- 都必须表现结果末尾的分页状态，至少包括上一页、当前页与下一页。
- 卡片点击只打开详情；详情主动作“使用此 LoRA”立即挂载并进入 Generate，行为不因方向改变。

三个方向必须在空间结构、第一视线、浏览节奏和信息揭示方式上明显不同；只更换颜色、圆角、卡片样式或控件皮肤不算一个新方向。本阶段不制作 Generate / Library / Train × 三方向的九套完整方案；先选定 Library 方向，再扩展整域身份并制作正式关键切片。

### 三个正式候选方向（已确认进入比较，未选向）

1. **聚焦浏览**：大幅效果预览主导，LoRA 按单列纵向节奏浏览，精确数据在用户选择后再展开。它用于验证最纯粹的“效果优先”，代价是单位屏幕内可扫视的模型更少。
2. **搜索工作台（当前推荐基准）**：搜索是主入口，结果使用克制的媒体网格，选中后由上下文检查器呈现详情与“使用此 LoRA”。它在高频浏览、精准选择和 Claude.ai / ChatGPT 式工作台秩序之间最平衡。
3. **对比工作台**：让多个候选及其样例、底模和触发信息形成并列比较。它适合专业选择，但信息密度最高，必须避免让数据重新压过效果。

“对比工作台”在本轮只允许组织当前已取得的结果与详情数据，不自动引入持久化候选集、批量操作或新的后端能力。三个方向将逐项确认内部结构后再做选择。

### Generate 现状审计入口（2026-07-19）

owner 提供并补充了同一桌面页面的五种必验状态，后续方向必须全部成立，不能只优化一张静态截图：

1. **未挂载 + 助手关闭**：纯底模可直接生成，同时提供去 Library 添加 LoRA 的辅助入口。
2. **已挂载 + 助手关闭**：展示当前装配、兼容底模、来源图/配方、提示词搭配、参考图输入和结果。
3. **未挂载 + 助手展开**：助手可以服务纯底模创作，但不能制造第二套页面骨架或遮蔽主台。
4. **已挂载 + 助手展开**：助手理解当前 LoRA、底模、触发词、提示词和参考图；展开后结果区仍保持可用。
5. **提示词搭配提醒**：推荐/自己搭配、触发词、结构化标签和助手建议需要形成清楚的提醒/应用状态，不能以当前独立大面板继续争夺主视线。

当前诊断：页面缺少在助手开关和挂载变化下仍稳定的“生成主台”。Generate 的结构讨论应先明确“当前装配 → 输入 → 参数 → 出图 → 结果”的主线，再决定来源配方和助手以何种辅助形态进入；旧 `docs/archive/design/lora-generate-bench-redesign-2026-07.md` 的一次性施工图不具备拍板权。

### Generate 逐项拍板（2026-07-19）

1. ✅ **稳定主台与辅助层边界已确认**：Generate 始终以“当前装配 → 输入与参数 → 出图 → 结果”为唯一主工作流。LoRA 挂载变化、纯底模状态和助手开关只能改变主台中的内容或辅助层状态，不得重排主工作流、制造第二套页面骨架，或让结果区失去可用尺寸。
2. ✅ **来源配方降为按需 modal 辅助层**：来源图位于主台上部；点击单张来源图后，在当前页面上方打开大尺寸 modal，主台保持原位并进入背景遮罩。modal 左侧固定展示大图，右侧独立滚动展示 Prompt、Negative Prompt 与完整参数；点击遮罩、按 Esc 或关闭按钮退出，不得跳转新页面、行内撑开主台或改变生成流程顺序。
3. ✅ **助手降为按需辅助层**：助手服务当前装配、输入与结果，不成为默认第一视线；展开后仍需保留主台的阅读顺序与操作连续性。
4. ✅ **五状态覆盖范围已确认**：未来方向必须同时展示未挂载、已挂载、两者各自的助手展开态，以及提示词搭配提醒；不能只画已挂载且助手关闭的默认态。
5. ✅ **来源图与参考图不得遗漏**：已挂载态需要展示 LoRA 的横向来源图证据；生成输入需要展示用户主动加入的参考图。两者语义不同，不能合并成一个模糊“图像”入口。
6. ✅ **底模与执行通道契约已确认**：挂载 LoRA 后自动选择其明确要求或推荐的兼容底模；应用来源配方后，若配方指定了更准确且真实可用的 checkpoint，则自动切换，并提供轻量原因说明与撤销/改选入口。“底模身份”与“执行通道”必须拆成两层；只有同一个真实底模同时具备兼容的 fal 与 Runner 路径时才显示通道切换。当前“心月狐”所属 Anima DiT 只有 Runner 路径，不能用 Flux LoRA 的 fal 路径伪装为替代通道。
7. ✅ **提示词搭配形态已确认**：删除当前常驻的“推荐 / 自己搭配”大型面板，改为紧贴生成输入区的单行搭配状态条；点击后在原位置展开差异摘要、标签检索/选择与冲突处理。LoRA 必需触发词自动加入并明确显示；来源配方、“做同款”或助手带来的 Prompt、Negative Prompt、底模和参数变化统一从该入口说明，提供“保留当前 / 使用配方 / 追加可兼容内容”与撤销，不使用阻断式 modal，也不得移动主台和结果区。
8. ✅ **Library / Generate 来源图详情共用 modal 已确认**：Library 行内详情和 Generate 装配区都保留位于上部的样例/来源图入口；点击单张图片后在当前页面打开“大图 + 完整结构化配方”的大尺寸 modal。modal 左侧固定展示图片，右侧独立滚动展示 Prompt、Negative Prompt、底模/checkpoint、尺寸、采样器、调度器、步数、CFG、Seed、标签与复制动作；点击遮罩、按 Esc 或关闭按钮退出，主台不重排。两处共享 dialog 行为、字段结构、焦点管理与响应式；Library 不因此恢复常驻配方块。Generate 版本增加“做同款”：只把真实可用的 Prompt、Negative Prompt、底模与参数应用到当前主台并关闭 modal，不直接发起付费生成；最终仍由用户检查后点击“出图”。已有输入时必须进入提示词搭配提醒，不能静默覆盖。owner 已确认桌面展开态，视觉证据见 [`references/pages/assets/lora-source-recipe-modal-desktop-2026-07.png`](../references/pages/assets/lora-source-recipe-modal-desktop-2026-07.png)。
9. ✅ **助手展开契约已确认**：桌面端使用约 360–400px 的右侧上下文边栏；主台顺序不变，并为结果区设置可用宽度底线，空间不足时改为覆盖式边栏而不是继续挤压。无 LoRA 时助手只读取纯底模、Prompt、Negative Prompt、参考图和参数，不伪造 LoRA 语义；已挂载时可以读取当前 LoRA、触发词、来源图配方、参考图、底模与参数。助手输出以可审阅变更卡呈现，只有用户点击“应用”后才进入搭配状态条；不得静默改写主台或直接出图。移动端使用近全屏 sheet，关闭后保留主台状态与滚动位置。
10. ✅ **用户参考图已添加态已确认**：撤回 64–72px 的紧凑缩略图行。参考图用于判断本次生成输入，已添加后在 Prompt 上方使用较大的预览卡横向展示；单图可预览/移除，末尾保留同尺寸“＋ 添加”，下方使用一个全局“参考强度”控制。点击添加仍复用现有上传、粘贴、最近素材与素材库选择能力。该视觉结论只约束参考图已添加态，不把整张上下文稿提升为 Generate 最终页面方向；确认图见 [`references/pages/assets/lora-generate-reference-images-added-desktop-2026-07.png`](../references/pages/assets/lora-generate-reference-images-added-desktop-2026-07.png)。
11. ✅ **Prompt 与参数空间层级已确认**：Prompt 是生成输入的主编辑面，桌面默认提供约 200–240px 的稳定编辑高度，并允许继续向下扩展；参考图区域不得把它压缩成短输入条。Negative Prompt 默认折叠，已有内容时在折叠标题中显示非截断语义摘要。生成参数使用 disclosure/折叠区而非布尔 Switch：收起时显示尺寸、Steps、CFG 等当前值摘要，展开后编辑采样器、调度器、Seed、尺寸与真实可用的后处理参数。展开只向下延伸或进入页面滚动，不反向挤压 Prompt。出图动作保持容易触达，不因高级参数展开而沉入不可见区域。
12. ✅ **用户参考图空态已确认**：未添加参考图时不预留已添加态的大卡片高度，只在 Prompt 上方显示一条低高度“添加参考图”入口，并简要说明可上传、粘贴、从最近素材或素材库选择。加入第一张后，原位置展开为第 10 项确认的大预览卡区域；删除最后一张后恢复紧凑空态。空态不显示无意义的参考强度，避免与 Prompt 争夺首屏空间。
13. ✅ **Generate 结构方向已选定 A「并排监视台」**：以“顶部轻量装配/来源证据 + 左侧完整生成输入 + 右侧稳定结果监视列”为关键切片基础。方向 B「居中工作单」和 C「装配脊柱」不进入关键切片，只保留为比较证据；B 的结果时间流只能表达刷新即清空的本次会话记录，不能称为长期资产；C 的多 LoRA 装配能力虽真实存在，但本轮不让常驻脊柱压过媒体结果。低保真方向图只确认空间骨架，不授权沿用其中偏小参考图、偏矮 Prompt、密集顶部 chrome 或错误结果动作；这些问题已在第 19 项的确认切片中修正。方向证据：[`assets/lora-generate-direction-a-desktop-2026-07.png`](assets/lora-generate-direction-a-desktop-2026-07.png) 与 [`assets/lora-generate-direction-a-state-map-2026-07.png`](assets/lora-generate-direction-a-state-map-2026-07.png)。
14. ✅ **Generate 顶部降噪结构已确认**：应用壳的“生成 / 库 / 训练”导航不计入业务装配层；导航下只保留一条无卡片、无外层面板的装配行，平铺当前 LoRA、权重、容量、底模身份、执行通道和助手入口。其下单独放置来源图带，已挂载时作为 LoRA 效果证据，未挂载时整行消失。取消“当前装配”等重复分组标题、装配卡套卡和额外 chrome；多挂载内容仍在同一装配行内表达，不因此新增常驻侧栏。

## 2. 当前 UI 真机诊断（2026-07-19）

在 `http://localhost:3000/zh/studio/lora` 桌面真机复核 Generate、Library、Train，并结合 owner 提供的详情层截图：

- **全域没有自己的第一眼识别物**：三处都依赖全站黑底、圆角容器、pill、细描边和同一字体，隐去顶部 LoRA 标签后很难判断属于哪个业务域。
- **Library 像通用素材浏览器**：顶层模式、公开/我的、来源、类型、底模形成多排同权重切换；图片内容有效，但筛选 chrome 与通用卡片网格压过了 LoRA 的模型差异、样例证据和使用决策。
- **Generate 像 Studio 生成页的变体**：挂载区、结果区与底部浅色 composer 空间关系松散；未挂载时大量空黑区域，LoRA 栈没有成为页面的视觉主角。
- **Train 像后台表单**：大圆角表单卡、预设卡和历史卡彼此独立，和 Library / Generate 没有可感知的域级亲缘关系。
- **详情层信息齐全但层级平均**：下载、喜欢、底模、触发词、标签、样例提示词和动作都落在相近的蓝黑容器语法里；用户能读到数据，却没有形成“为什么值得用、效果证据是什么、下一步做什么”的强节奏。

因此本轮目标是**保留功能骨架，重做视觉层级与域级身份**，不是继续补功能，也不是给现有组件统一换颜色。
