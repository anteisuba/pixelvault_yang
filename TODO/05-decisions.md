# 05. 需要你拍板的关键决定

这些决定我已经先给了建议值。后面我们真的开始构建时，你只需要按这份清单逐个确认就行。

## 1. 第一版开放方式

- 建议：公开可访问，但生成能力先用注册用户 + 配额控制。
- 原因：可以传播，也能控成本。

## 2. 默认公开还是默认私密

- 建议：图片默认私密，用户手动发布到 Gallery。
- 原因：这更符合真实用户对作品和 Prompt 的预期。

## 3. Prompt 是否参与推荐

- 建议：默认不参与，用户明确同意后才进入推荐统计。
- 原因：这样更安全，也更适合以后公开发布。

## 4. 第一版要不要做支付

- 建议：不要。
- 替代方案：免费 Beta、注册送额度、每日额度、邀请码。

## 5. 第一批接多少模型

- 建议：3 到 5 个。
- 原因：足够让用户感知“可切换”，但不会把维护成本拉爆。

## 6. 要不要支持用户自带 API Key

- 建议：第二阶段再做。
- 原因：第一版先把平台模式跑通，BYOK 会把鉴权、加密、计费、体验都变复杂。

## 7. 生成流程要不要马上上队列

- 建议：数据结构现在就按队列化设计，执行层第一版可以先同步。
- 原因：这样节奏更快，但不堵死后路。

## 8. Prompt 推荐做成什么样

- 建议：模板库 + 热门意图卡片。
- 不建议：直接公开别人的完整 Prompt 榜单。

## 9. 手机端怎么推进

- 建议：先响应式 Web，再考虑 PWA，最后再决定原生 App。
- 原因：这是投入产出比最高的路线。

## 10. 我建议你现在就默认采用的方案

- 产品路线：Web Beta
- 账号策略：登录后生成
- 可见性：私密默认，发布可选
- Prompt 策略：私有历史 + 同意后统计 + 模板化推荐
- 商业化：先不支付，先账本
- 架构：Provider Adapter + 异步就绪
