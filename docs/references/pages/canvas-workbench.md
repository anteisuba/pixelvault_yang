## 1. 域职责与本页任务（详见 domains/canvas.md，此处不重复）

长视频导演台：剧本→分镜→逐镜生成→拼接。本页（`/studio/node`）承载全部六条核心工作流；最高频任务 = **一组素材汇入一个视频镜头**。非目标：不复制 Studio 工作台、不做通用白板、不做 ComfyUI 式接线板。

## 2. 设计性格与禁区（拍板）

**「片场的掌控感」**：素材是主角（画面发色，chrome 退后）· 叙事连续感（任何时刻答得出"我的片子现在什么样"）· 决断反馈（每个动作快、小、准、可撤销）。

禁区：不像参数表单页（Studio）；不像技术接线板（端口/管线情绪）；不像白板 SaaS（便签/马克笔情绪）；chrome 永不压过媒体；不支持的能力不渲染。

## 4. 三个标志性组件

### 4.1 镜头单元（媒体即卡面 + 成分坞）

- **帧即卡**：媒体 edge-to-edge 铺满卡面，**2px 直角**（crisp 取景感，无纸卡 chrome、无内边距裱框）。
- **信息条**：压帧下缘的半透明 scrim（高 ~22px）：镜号（mono）· 名 · 时长/状态（右对齐 mono）。空态无媒体时信息条仍在。
- **成分坞**：熔接在帧左缘的竖轨——喂进本镜的一切以缩略常驻（出场脸 22px 方块 / 场景缩略 / 音色波形格 / 参考视频格）+ 尾部虚线添加位。**归属靠"长在一起"表达，取代吞噬的消失**；坞项可点（浮出来源）可拖出（解绑）。
- **状态**：空 = 虚线帧 + 生成 glyph；选中 = **琥珀取景框 2px + 四角标**；生成中 = REC 红点 + 帧内扫描线（唯一持续动效，对应真实后台态）；失败 = 红框 + 原因条；就绪/完成 = 中性，信息条状态词表达。
- 尺寸：帧逻辑宽沿用现有节点宽度 token 体系；坞宽 ~26px。

### 4.2 身份名册牌

- 角色 = 44px 头像方块 + 名（下置）；场景 = 横缩略 + 名；音色 = 波形药丸 + 名。极简立牌，无卡体 chrome。
- 一牌喂多镜 = 一致性复用（数据层收割/出场组/合法矩阵原样）。
- **系带线**：仅当某镜头被选中时，从其成分对应的名册牌浮现 1px 中性细线到坞（答"这张脸是谁"）；平时桌面零线。
- 名册牌详情（图集/出场组/绑音）沿用既有档案能力，入口 = 双击/工具条，视觉随新基调重铺。

### 4.3 成片托盘

- 底部居中常驻、可收拢：收拢 = 细条「成片 · N 镜 · 总长(mono)」；展开 = 槽位缩略 + 每槽时长 + 尾部虚线添加位。
- 槽位顺序 = 片序（**序列不用线画，位置即顺序**）；当前选中镜对应槽琥珀框；拖镜头单元入托 = 入序。
- 托盘背后即片盒/合成业务（videoMerge），行为契约不变。

## 5. 域 token 层（strangler 新组，旧 `--node-*` 迁完即删）

新组前缀 `--canvas-stage-*`（避开既有 `--canvas-*` 几何变量与旧 `--node-*` 皮肤）：

| Token                         | 值                   | 用途                                                 |
| ----------------------------- | -------------------- | ---------------------------------------------------- |
| `--canvas-stage`              | `#101114`            | 舞台（冷石墨，偏蓝黑——不给画面加暖偏，素材发色最准） |
| `--canvas-stage-grid`         | `#1E222A`            | 点阵（存在感极低）                                   |
| `--canvas-stage-panel`        | `#14171B`            | chrome 面板（托盘/坞/顶条/助手签）                   |
| `--canvas-stage-panel-border` | `#262B31`            | 面板描边                                             |
| `--canvas-stage-well`         | `#1D2126`            | 媒体占位/空帧底                                      |
| `--canvas-stage-ink`          | `#E7E9EC`            | 主文字                                               |
| `--canvas-stage-ink-muted`    | `#8B919A`            | 次级文字                                             |
| `--canvas-stage-ink-subtle`   | `#6B727C`            | 提示/占位                                            |
| `--canvas-stage-scrim`        | `#0C0D10 @ 82%`      | 信息条底                                             |
| `--canvas-stage-signal`       | `#E8A33D`            | **钨丝琥珀单信号**：选中取景框/托盘当前槽/主行动键   |
| `--canvas-stage-signal-ink`   | `#101114`            | 琥珀底上的字                                         |
| `--canvas-stage-tether`       | `#3A4048`            | 系带线/虚线帧                                        |
| REC/错误                      | 复用全局 danger 语义 | 生成中红点/失败框                                    |

字体：UI 走全局 sans；**镜号/时长/时码/总长一律 mono + tabular-nums**（片场仪器感的最便宜来源）。圆角：帧 2px 直角、面板 6px、药丸仅音色格。对比度：实现时确定性脚本复核（正文 ≥4.5:1、UI 边界 ≥3:1），信号色只作边框/小面积底不作正文色。信号色纪律：**一屏唯一强调色**，禁第二支彩色；REC 红仅真实生成中/错误。

## 6. 关系表达契约（取代旧吞噬/两级墨线）

| 关系              | 表达                                                                                                           | 数据层                 |
| ----------------- | -------------------------------------------------------------------------------------------------------------- | ---------------------- |
| 供给（素材→镜头） | **成分坞常驻缩略**（归属=长在一起）；选中时名册牌系带线浮现                                                    | 仍是边；坞=渲染投影    |
| 序列（镜头→成片） | **托盘槽位位置**                                                                                               | 仍是 videoMerge 边     |
| 绑定手势          | 拖素材/名册牌**到帧或坞** = 入坞吸附（快、小、准的落位动效）；拖出坞 = 解绑；磁吸/快投等行为层能力保留语义重接 | 建边/删边路径不变      |
| 退役              | 常驻连线、端口、吞噬三拍"消失"动画、两级墨线渲染、关系线开关                                                   | 合法性矩阵与图模型原样 |

素材原子不再"被吞消失"：散图被吃 = 缩入目标坞（可视迁移动效），本体从桌面移除但**坞里永远看得见、拖得出**。

## 7. 页面结构与关键状态

**桌面**：全幅舞台；顶条极薄（项目名 · 添加 · 缩放）；左侧无固定栏（名册牌自由摆，可框选群移）；底部成片托盘；右缘助手薄签（展开为侧栏，行为沿用现契约）。
**移动**：同一舞台直排；托盘沉底可展开；添加为浮钮；助手全屏抽屉（ResponsiveOverlay 契约）。
**关键状态**：空画布起手（虚线首帧 + 起手动作，不留死空白）；装配中（坞渐满）；生成中（REC+扫描，可离开回来恢复）；失败（红框+原因+重试）；托盘空（细条提示）；`prefers-reduced-motion` 全降级瞬时。

## 8. 不能长得像谁

| 域/物              | 差异来自                                                             |
| ------------------ | -------------------------------------------------------------------- |
| Studio Image/Video | 那边是表单→结果；这边是空间编排+序列，无长表单                       |
| LoRA 工作台        | 那边是资源库+装配栈；这边媒体即卡面+托盘脊柱                         |
| ComfyUI/接线板     | 无端口无常驻线，归属靠坞、序列靠托盘                                 |
| 白板 SaaS          | 对象是有类型的制片单元非便签；单信号色+mono 仪器感非马克笔           |
| 旧暗炭制片桌       | 冷石墨 vs 暖炭；帧直角 vs 纸卡圆角；琥珀 vs 石绿；坞 vs 吞噬——全断开 |

## 9. 继承的全局行为（brand-dna 品质底线 + 共享组件契约）

键盘可达/焦点环/ARIA；命中区 fine 32/36 · coarse 44；ResponsiveOverlay/触屏软键盘/focus return；i18n 三语；状态真实不伪装；共享组件复用行为与 API，皮肤由本页 token 覆盖。Del 解绑、双击开详情、Esc 逐层退、多选合成等**行为**沿用已落地契约。

## 10. 实现切片建议（交执行，一次一片全绿）

1. **C1 token 舞台**：`--canvas-stage-*` 组落地 + 舞台/点阵/顶条换皮 + 对比度脚本复核。
2. **C2 镜头单元**：帧即卡 + 信息条 + 成分坞（渲染投影既有边）+ 四状态 + 选中取景框。
3. **C3 名册牌**：三型立牌 + 系带线（选中显现）+ 详情入口重铺。
4. **C4 成片托盘**：收拢/展开 + 槽位序 + 拖入入序 + 与 videoMerge 行为对接。
5. **C5 chrome 收尾**：助手薄签/移动直排/空态起手 + 旧 `--node-*` 皮肤引用清理 + 视觉基线重建。
   每片：lint + 全量 tsc/vitest + 真机目检；数据层零改动为硬验收。
