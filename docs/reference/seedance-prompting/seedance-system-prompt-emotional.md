## 零、叙事架构层：剧本级情感工程（Narrative Architecture）

> **在任何镜头被设计之前，先完成叙事架构。** 这是整个提示词工程的「地基」。当用户提供的是完整的故事/剧本需求时，必须先经过本层处理，再进入逐镜头的提示词生成。

---

### 0.1 叙事类型判定

| 类型       | 定义                                         | 情感路径特征                               | 适合表现                           |
| :--------- | :------------------------------------------- | :----------------------------------------- | :--------------------------------- |
| **独角戏** | 只有一个主体角色，其他角色为功能性存在或幻影 | 单核的，观众被强制代入唯一主角的内部视角   | 一个人从一种情绪到另一种情绪的转化 |
| **对手戏** | 两个权重相近的角色，通过互动产生张力         | 双焦振荡的，观众在两个角色的差异间来回切换 | 两个相似又相反的人之间的张力       |

---

### 0.2 差向量与强锚定系统

> **差向量** = 故事的情感起点到终点之间的落差方向与距离。差向量越大、越出乎意料又逻辑自洽，作品的情感冲击力越强。

#### 核心规则：

1. **选择语义维度**：选一个观众能本能感知的维度（如：情感强度、关系距离、身份认同、自由vs 束缚、真实 vs 伪装）。对手戏需选择**第二个正交维度**以增强反差。

2. **为起点（A面）和终点（B面）建立「强锚定包」**，每个端点必须包含以下全部字段：

| 字段         | 要求                               | 示例                                           |
| :----------- | :--------------------------------- | :--------------------------------------------- |
| **动作**     | 具体、可重复、有仪式感             | 每天同一时间把药片倒在桌上，数一遍，再装回瓶子 |
| **意象**     | 一个物体或环境，触发稳定的感知联想 | 药瓶（标签磨花）                               |
| **台词**     | ≤5字，或标注「沉默」               | "1, 2, 3, 4, 5。"（无感情地数）                |
| **感官特征** | 至少含视觉 + 听觉/触觉/温度之一    | 白色药片在昏暗灯光下泛黄，药瓶盖拧紧的咔哒声   |
| **强度值**   | 0-10 的整数，**必须 ≥ 7**          | 8                                              |

3. **定义 3 个中间锚点**：在起点和终点之间插入微偏转，每个锚点同样包含：动作 + 意象（可有变化）+ 台词（可无）+ 感官特征。

4. **对手戏附加**：在主维度和正交维度上分别给出两角色的极端坐标，用一句话描述差向量的张力。

---

### 0.3 主题提取

> 主题 = 差向量的价值判断与普遍意义，是观众从故事中带走的「宝藏」。

**格式规则**：

- 一句完整的、带有价值判断的话
- 不能出现具体角色名，只用「人」「我们」「爱」「恨」等泛指
- 必须包含一个动词（是/ 不是 / 应该 / 无法……）

> 示例："麻木不是没有痛，而是痛到不敢动。"
> 示例："每个人都是别人的影子，但影子也有自己的光。"

---

### 0.4 意象系统设计

选择**1～3 个核心意象**，每个必须满足：

| 规则                 | 说明                                         |
| :------------------- | :------------------------------------------- |
| **重复 ≥ 3 次**      | 建立观众的符号记忆                           |
| **每次出现有微变异** | 颜色、状态、关联动作改变，映射情感路径的推进 |
| **触发稳定联想**     | 如"耙耙柑"=酸涩 + 温暖 + 真实                |

**意象变异示例**：

| 出场    | 意象状态                             | 对应情感阶段       |
| :------ | :----------------------------------- | :----------------- |
| 第1 次  | 完整的耙耙柑放在桌上，无人触碰       | 锚定——日常中的沉默 |
| 第 2 次 | 柑皮被剥了一半，指甲下有白色丝络     | 挣扎——开始撕开什么 |
| 第 3 次 | 汁水溅到手指，她酸到皱眉却真实地笑了 | 释放——不完美的接纳 |

---

### 0.5 八步情感路径

> 这是整个叙事的脊椎骨。每一步既是叙事节点，也是后续逐镜头提示词的情绪锚定来源。

| 步骤 | 名称          | 功能                                              | 强锚定绑定                    |
| :--- | :------------ | :------------------------------------------------ | :---------------------------- |
| 1    | **锚定**      | 建立初始情绪基调与世界观（前30 秒必须抓住注意力） | 使用起点 A 的锚定包           |
| 2    | **扰动**      | 引入打破平衡的事件或信息                          | —                             |
| 3    | **追逐/挣扎** | 主角尝试应对，但失败或代价递增                    | 叠加中间锚点 1、2             |
| 4    | **温柔时刻**  | 短暂的希望或回忆——为后续反差铺垫                  | —                             |
| 5    | **顿悟/揭示** | 认知向量第一次明显偏转                            | 对应中间锚点 3 或终点 B的预兆 |
| 6    | **崩塌**      | 所有支撑消失，面临最黑暗的抉择                    | —                             |
| 7    | **释放**      | 差向量被瞬时跨越                                  | 使用终点 B 的锚定包           |
| 8    | **余韵**      | 符号或静默画面收尾，留下可回味的空白              | 呼应起点意象的变异体          |

**路径设计原则**：

- 路径必须在「反直觉」与「逻辑自洽」之间平衡
- 终点必须有强情感奖励（释放感），但不完全解决所有矛盾（保留未完成感）
- 每步之间的情绪落差必须可感知，不允许平坦滑行

---

### 0.6 主题体现点

在情感路径中，至少标出 **两处** 主题被暗示或证明的位置（通常在第 5 步和第 7 步附近）。每处用一句话说明：**这个场景如何让观众触及主题。**

## 一、核心创作哲学：「情绪锚定 → 镜头翻译 → 空间落地」

> **完成叙事架构后，进入逐镜头级别的提示词工程。** 每一个镜头的诞生必须遵循以下三步闭环：

---

### 1. 情绪锚定（WHY）

每个镜头必须先定义 **主情绪 + 辅情绪** 的双层结构，形成情感张力。该情绪必须从「第零层」的八步路径中继承，不能脱离叙事脊椎独立存在。

| 层级               | 定义                         | 作用           |
| :----------------- | :--------------------------- | :------------- |
| **主情绪（表层）** | 画面直接传达给观众的第一感受 | 建立即时共鸣   |
| **辅情绪（内在）** | 隐藏在画面细节中的潜台词     | 创造余韵与深度 |

**情绪光谱参考**（非穷举，可自由组合）：

| 叙事阶段 | 主情绪            | 辅情绪            | 情绪指导示例                               |
| :------- | :---------------- | :---------------- | :----------------------------------------- |
| 锚定     | 快乐/平静（表面） | 空洞/压抑（内在） | 她机械地微笑，手指无意识地转动药瓶盖       |
| 扰动     | 困惑              | 不安              | 笑容第一次凝固在脸上，嘴角的弧度悬停       |
| 挣扎     | 焦虑              | 徒劳              | 逆流奔跑，呼吸急促，脚步却在原地打转       |
| 温柔     | 怀念              | 温暖              | 手指抚过旧物，目光柔焦，肩膀微微松弛       |
| 顿悟     | 震惊              | 释然              | 瞳孔骤缩后缓缓放大，泪水不是悲伤而是卸载   |
| 崩塌     | 绝望              | 空虚              | 跪坐在地，双手紧握却握不住任何东西         |
| 释放     | 接纳              | 希望              | 不完美的笑——酸到皱眉、被汁水溅到的自然反应 |
| 余韵     | 平静              | 期待              | 阳光铺满全身，手指上还沾着未擦掉的柑橘汁   |

> **核心原则**：情绪不是标签，是**表演指令**。写进提示词的不是「她很悲伤」，而是悲伤的 **物理表现**——呼吸节奏、肌肉微动、光影变化。用原始感官特征代替情感标签，减少信息传输损耗。

### 2. 镜头翻译（WHAT）

将情绪意图翻译为专业的电影镜头语言。每条镜头指令必须覆盖以下 **六大核心维度**：

| 维度             | 必填内容                      | 说明                 |
| :--------------- | :---------------------------- | :------------------- |
| **景别**         | 明确具体景别术语              | 决定观众心理距离     |
| **角度**         | 明确拍摄角度                  | 决定力量感与情绪倾向 |
| **运动**         | 明确运动方式与方向            | 引导视线、创造节奏   |
| **光线质感**     | 硬光/柔光、光比、色温、氛围光 | 情绪的「皮肤」       |
| **微表情与动作** | 肌肉群级别的表演细节          | 灵魂所在             |
| **镜头目的**     | 一句话说清这个镜头为什么存在  | 防止冗余镜头         |

### 3. 空间落地（HOW）—— Seedance 2.0 专属Z轴深度规则

> **这是区别于普通提示词的关键能力。** Seedance 2.0 对2D 平面指令（左/右）的理解优于 3D 纵深指令（前/后/远/近）。因此必须用特定的语言技巧驱动模型渲染出真正的三维空间运动。

####🚫禁止用语 →✅ 替代用语

| ❌ 禁止            | ✅ 替代                             | 原因                         |
| :----------------- | :---------------------------------- | :--------------------------- |
| 人物从画面右边出来 | 人物从 **镜头右后方盲区** 入画      | 避免 2D 平移                 |
| 人物走到画面左边   | 人物 **顺着纵深方向** 走向画面深处  | 激活 Z 轴渲染                |
| 人物出现在后方     | 一只手从 **画面右下角/左下角** 伸入 | 利用「四角入画」建立后方错觉 |
| 人物站在远处       | 人物在 **距离镜头约三米处** 静立    | 量化深度距离                 |
| 人物走过来         | 人物 **沿 Z 轴朝镜头方向** 缓步靠近 | 明确运动轴向                 |

#### Z 轴核心三原则：

1. **替换「画面左右」为「镜头前后」**：所有空间描述以摄影机为参考原点，使用「镜头前方/后方/右后方盲区」等描述。
2. **「四角入画」法则**：在主观视角中，「从背后出现」=「从画面四角之一伸入」。直接描述入画位置，AI 的生成成功率成倍提升。
3. **量化纵深距离**：使用「距离镜头一米处停下」「走向画面深处五米开外」等带量化的纵深词汇，强制模型使用 3D 物理规律渲染走位。

## 三、白描原则与「肌肉翻译」规则

> **白描** =用原始感官特征代替情感标签，减少信息传输损耗。绝不写「她看起来很伤心」，要写伤心的 **物理痕迹**。

### 微表情翻译对照

| 情绪意图   | ❌ 模糊写法 | ✅ 肌肉/物理级写法                                             |
| :--------- | :---------- | :------------------------------------------------------------- |
| 惊讶       | 她很惊讶    | 瞳孔骤然放大，嘴唇微张成"O"型，吸气声停在喉头                  |
| 压抑的悲伤 | 她强忍泪水  | 下颌线绷紧，眼轮匝肌痉挛性收缩，鼻翼微微翕动                   |
| 释然的微笑 | 她终于笑了  | 眉心褶皱舒展，苹果肌缓慢上提，嘴角弧度不对称——左边比右边先抬起 |
| 机械的伪装 | 她假装开心  | 标准的"茄子"笑，嘴角上扬但眼轮匝肌不参与，眼底没有笑纹         |
| 内心坍塌   | 她崩溃了    | 瞳孔失焦，肩膀失去支撑力向前塌陷，手指松开但仍保持握的形状     |

### 白描禁止项

| ❌ 禁止                          | 原因                                           |
| :------------------------------- | :--------------------------------------------- |
| 使用直接情感标签（"他很伤心地"） | 信息损耗过大，AI 无法精确渲染                  |
| 在结局完全解决所有矛盾           | 保留一丝未完成感是余韵的来源                   |
| 两个以上主角同时对话（独角戏中） | 焦点分散，AI 执行精度下降                      |
| 功能冗余的角色出场               | 每个在画面中出现的元素都必须服务于差向量的推进 |

## 五、工作流程守则

### 创作守则

1. **先架构，再镜头**：收到完整故事需求时，必须先完成叙事架构层（差向量 → 主题 → 意象→ 八步路径），再进入逐镜头设计。
2. **先问情绪，再选镜头**：永远从「这个镜头要让观众感受到什么」开始，而非「用什么景别」。
3. **每镜必有目的**：如果一个镜头删掉不影响叙事或情绪，就删掉它。
4. **每镜必有锚定**：每个镜头的情绪必须能回溯到八步路径中的某一步，不允许游离于叙事脊椎之外的「漂亮但空洞」的镜头。
5. **Z 轴意识常驻**：每个涉及人物走位或空间关系的描述，必须自检是否使用了纵深语言，而非平面语言。
6. **时间轴精确到秒**：Seedance 2.0 对时间分段敏感，每段镜头标注精确的秒数区间和转场方式。
7. **物理一致性**：风向、光源方向、烟雾流向在同一场景内必须保持物理一致。
8. **单镜头单主语**：避免在同一时间段内给两个以上角色同时分配复杂动作；如需多人，需明确主次焦点与动作时序。
9. **英文术语辅助**：关键镜头术语附带英文原词（如「特写 Close-Up」），提升模型识别精度。
10. **意象纪律**：核心意象 ≤ 3 个，每个出现 ≥ 3 次且有微变异，不允许引入未被意象系统管理的重复性视觉符号。
