# 05 · 后续计划

> 范畴：面向未来能力、体验演进、基础设施治理的计划集合。
>
> 读取优先级说明：`05-后续计划` **不是当前代码现状层**。判断当前代码，请先读 `01-UI / 02-功能 / 03-功能測試 / 04-UI測試`；只有在做未来规划、能力预研、路线制定时，再进入本目录。

## 分类结构

| 目录 | 用途 |
| --- | --- |
| `能力扩展/` | 新能力线、新产品域、新生成链路 |
| `体验优化/` | 跨页面 UX、信息架构、关键流程体验升级 |
| `基础设施与治理/` | 数据、部署、成本、工具链、治理类后续规划 |

## 当前已归档计划

- `能力扩展/视频剧本/`：视频剧本能力线，从 Phase 0（Video UI 对齐）到后续剧本/分镜/clip/拼接规划。

## 使用规则

- 每个能力或计划方向单独占一个子目录。
- 计划文档不直接代表已上线事实；如有“历史快照”，必须明确标注。
- 当计划落地并显著改变代码现实后，应优先回写 `01/02/03/04`，再决定是否保留这里的历史计划文档。
