# Docs Directory

`docs/` contains the current project documentation, historical plans, and
durable reference material for Personal AI Gallery.

## Directory Guide

- `api/`
  - provider notes and external API references
- `architecture/`
  - system-level structure and data flow
- `backend/`
  - server-side implementation notes
- `database/`
  - Prisma and data model references
- `design/`
  - design analyses and external inspiration
- `frontend/`
  - frontend design system and component references
- `guides/`
  - durable operating rules and process guidance
- `plans/`
  - exploration output and implementation plans
- `product/`
  - roadmap, scope, product direction
- `progress/`
  - current project status and phase tracking
- `qa-screenshots/`
  - visual snapshots from QA passes
- `tooling/`
  - AI workflow and project tooling notes

## Start Here

If you need the current implementation state, read these first:

- `architecture/system-architecture.md`
- `progress/phases.md`
- `progress/current-status-audit.md`
- `product/roadmap.md`
- `plans/product/unified-development-plan.md`

If you specifically need Studio:

- `plans/frontend/studio-feature-map.md`
- `plans/frontend/studio-workbench-redesign.md`
- `plans/frontend/studio-v3-redesign.md`

## Update Rules

- keep current-state docs aligned with the latest code
- keep historical plan docs, but clearly mark when they are no longer the source of truth
- add new exploration output to `docs/plans/`
- add durable rules/process docs to `docs/guides/`
- avoid mixing temporary notes into architecture or roadmap docs
